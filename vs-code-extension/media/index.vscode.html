VS Code Help

Basics for first timers

  • Open the vs-code/ folder in VS Code, press F5 to start the Extension Development Host, then run “Modern Editor: Open” in that host’s Command Palette.
  • Open the Modern Editor panel and click “Load from VS Code” to pull the active file (selection and version included).
  • Edit in the webview, then use “Save & Compile” to push changes and run LaTeX Workshop build/view (applies at the loaded version/selection when available; we try to preserve focus).
  • During Save & Compile and SyncTeX, VS Code may briefly focus the LaTeX/PDF tab; return to Modern Editor afterward. Alt/Option+click forward-syncs; Cmd/Ctrl+click reverse-sync stays in the PDF viewer.
  • Use the hamburger menu for apply/load, sessions, and supporting files.

Modern Editor (VS Code)

Private/local VS Code extension that hosts Modern Editor in a webview.

Dev run

  1. Open vs-code/ in VS Code.
  2. Run Modern Editor: Open (Extension Development Host).

Notes

  • All extension assets live under vs-code/.
  • Webview assets are under vs-code/media/.
  • Logging goes to the “Modern Editor” Output channel.
  • Supporting file uploads are not yet supported in the VS Code extension.
  • Use the hamburger menu to “Load from VS Code” / “Apply to VS Code”.
  • LaTeX Workshop integration: “Save & Compile” runs latex-workshop.build then latex-workshop.view.

Help

Super Spell/Style/Grammar Check is a local editor that sends your text to a selected AI model for feedback and suggested edits.

Quick start

  1. Paste or upload a document, or select a passage to focus.
  2. Choose a model and reasoning effort in the menu, then run Grammar, Style, Everything, or Custom.
  3. Review suggestions: use left/right arrows to move, Enter to accept, Backspace/Delete to reject.

Tools

  • Grammar / Style / Everything: Rule-based passes over the selection or full document.
  • Custom: Edit instructions, scope, and aggressiveness; start from a preset if helpful.
  • Simplify: Creates three simpler rewrites of a selected passage.
  • Proof: Beta logic check for a selected theorem and proof.
  • Custom Ask: Ask a question about selected text; returns comments plus suggestions.

Selections include a small read-only context window. You can optionally add the full document as an extra read-only context message (costly). Full-document runs include the LaTeX preamble.

Files and history

  • Upload: Replace the document or add supporting files (PDF, markdown, LaTeX).
  • Supporting files: Sent with your prompt and count toward cost.
  • Run history: Optional local storage in this browser profile.
  • Sessions: Save or load snapshots from the menu.
  • Request timeout: Optional menu setting; Off by default. Use it if a run hangs.

Privacy and safety

  • Data: Selected or full text is sent to the chosen model provider.
  • Local storage: Sessions, run history, and file metadata may be saved locally. API keys can be stored in VS Code Secret Storage.
  • Sensitive data: Avoid if you cannot send it to an external provider.
